# MBTI 성격 유형 테스트 웹사이트 PRD

## 1. 프로젝트 개요

### 1.1 프로젝트 소개
MBTI 성격 유형 테스트 웹사이트는 사용자가 간단한 질문에 답변하여 자신의 성격 유형을 파악할 수 있는 웹 기반 서비스입니다. 12개의 질문을 통해 16가지 MBTI 유형 중 하나를 판별하고, 해당 유형의 특징과 정보를 제공합니다.

### 1.2 프로젝트 목표
- **접근성**: 누구나 쉽게 접근하고 5분 이내에 완료할 수 있는 간편한 테스트 제공
- **몰입도**: 직관적이고 매력적인 UI/UX를 통해 80% 이상의 테스트 완료율 달성
- **확산성**: 결과 공유 기능을 통한 자연스러운 바이럴 효과 창출
- **참여도**: 실시간 통계 데이터를 통해 사용자 참여와 재방문 유도
- **신뢰성**: 정확하고 신뢰할 수 있는 MBTI 유형 분석 결과 제공

### 1.3 타겟 사용자
- **주요 타겟**: 자기 이해에 관심 있는 10대~30대 사용자
- **세부 타겟**:
  - MBTI 및 성격 테스트에 관심이 많은 사용자
  - 소셜 미디어를 활발히 사용하는 사용자
  - 심리 테스트 및 자기계발 콘텐츠를 즐기는 사용자
  - 친구들과 결과를 공유하고 비교하고 싶어하는 사용자

### 1.4 사용자 니즈
- 빠르고 간편한 테스트 경험 (모바일 환경에서도 원활한 진행)
- 시각적으로 매력적이고 공유하기 좋은 결과 페이지
- 손쉬운 결과 공유 기능 (SNS 연동)
- 신뢰할 수 있고 구체적인 결과 분석
- 다른 사람들의 테스트 결과와 비교 가능한 통계 정보

## 2. 핵심 기능 요구사항

### 2.1 테스트 질문 시스템

#### 2.1.1 질문 구성
- **총 12개 질문** (각 MBTI 차원당 3문항)
  - E/I (외향성/내향성): 3문항
  - S/N (감각/직관): 3문항
  - T/F (사고/감정): 3문항
  - J/P (판단/인식): 3문항
- 각 질문은 일상적이고 공감 가능한 상황 기반으로 구성
- 답변은 2지선다형 (A 또는 B)

#### 2.1.2 질문 내용 방향성

**E/I 차원 (에너지 방향):**
- 사회적 상황에서의 선호도
- 에너지 충전 방식
- 대인관계 패턴

**S/N 차원 (인식 방식):**
- 정보 처리 방식
- 문제 해결 접근법
- 미래 vs 현재 지향성

**T/F 차원 (판단 기준):**
- 의사결정 기준
- 타인의 고민에 대한 반응
- 가치 판단 방식

**J/P 차원 (생활 양식):**
- 계획성 vs 즉흥성
- 업무 진행 방식
- 일상생활 패턴

#### 2.1.3 답변 방식
- 각 질문당 2개의 선택지 제시
- 사용자가 답변 선택 시 즉시 다음 질문으로 자동 이동 (선택 사항)
- 이전 질문으로 돌아가 답변 수정 가능
- 현재 진행 상황을 시각적으로 표시 (프로그레스 바)

#### 2.1.4 결과 산출 방식
- 각 차원별로 점수를 집계
- 각 차원에서 더 높은 점수를 받은 선호를 조합하여 4자리 MBTI 코드 생성
- 예: E(3점) vs I(0점), S(1점) vs N(2점), T(2점) vs F(1점), J(0점) vs P(3점) → ENTP

---

### 2.2 진행 상황 표시

#### 2.2.1 프로그레스 바
- 전체 질문 대비 현재 진행률을 시각적으로 표시
- "질문 N/12" 형태로 텍스트 병기
- 답변 선택 시 부드러운 애니메이션으로 진행률 증가
- 페이지 상단에 고정 배치하여 항상 보이도록 함

---

### 2.3 결과 페이지

#### 2.3.1 표시할 정보

**1. MBTI 유형 헤더**
- MBTI 4자리 코드 (예: ENFP)
- 유형의 별명 (예: "재기발랄한 활동가")

**2. 성격 특징 개요**
- 해당 유형에 대한 전반적인 설명 (2-3 문단)
- 핵심 특징 3-5개를 불릿 포인트로 요약

**3. 강점 섹션**
- 해당 유형의 주요 강점 5-7개 나열
- 각 강점에 대한 간략한 설명

**4. 약점 섹션**
- 주의해야 할 약점 5-7개 나열
- 개선 방향이나 극복 방법 제시

**5. 추천 직업**
- 해당 유형에 적합한 직업군 8-10개
- 카테고리별로 그룹화 (예: 창작, 경영, 교육, 서비스 등)

**6. 같은 유형의 유명인**
- 같은 유형으로 알려진 유명인 5-8명
- 국내외 유명인 혼합

#### 2.3.2 결과 페이지 레이아웃 구성
```
┌─────────────────────────────────┐
│      [MBTI 타입 - 큰 글씨]      │
│         [유형 별명]              │
├─────────────────────────────────┤
│     [성격 특징 개요 및 설명]    │
├─────────────────────────────────┤
│      [강점]  │    [약점]        │
├─────────────────────────────────┤
│         [추천 직업]              │
├─────────────────────────────────┤
│        [유명인 목록]             │
├─────────────────────────────────┤
│  [액션 버튼들: 저장/공유/재시작] │
├─────────────────────────────────┤
│         [통계 차트]              │
└─────────────────────────────────┘
```

### 2.4 공유 기능

#### 2.4.1 이미지 저장
- 결과 페이지를 이미지 파일로 저장하는 기능
- SNS 공유에 최적화된 이미지 비율 (1:1 또는 16:9)
- 고해상도 이미지 생성 (레티나 디스플레이 대응)
- 다운로드 파일명: "my-mbti-[타입].png" 형식

#### 2.4.2 결과 공유
- 모바일 브라우저의 네이티브 공유 기능 활용 (Web Share API)
- 공유 내용:
  - 제목: "나의 MBTI는 [타입]!"
  - 설명: "나는 [유형 별명]입니다!"
  - URL: 현재 페이지 링크
- Web Share API 미지원 브라우저를 위한 대체 기능:
  - URL 복사 버튼
  - 복사 완료 시 알림 메시지 표시

#### 2.4.3 SNS 직접 공유 (선택 사항)
- 주요 SNS로 직접 공유할 수 있는 버튼 제공
- 지원 플랫폼: 카카오톡, 페이스북, 트위터, 링크 복사

---

### 2.5 통계 기능

#### 2.5.1 수집할 데이터
- 각 MBTI 유형별 테스트 완료 횟수
- 전체 테스트 참여자 수
- 유형별 백분율 분포

#### 2.5.2 통계 표시 내용
- **전체 테스트 참여자 수** 표시
- **16가지 유형별 분포**를 시각화 (바 차트 또는 도넛 차트)
- 각 유형별 백분율과 실제 참여자 수 병기

#### 2.5.3 데이터 저장 방식
- 브라우저 로컬 스토리지 활용 (클라이언트 사이드)
- 사용자의 개인정보는 수집하지 않음
- 익명 통계 데이터만 저장

### 2.6 기타 기능

#### 2.6.1 다시 테스트하기
- 결과 페이지에서 테스트를 처음부터 다시 시작할 수 있는 버튼
- 이전 답변 초기화 후 첫 질문으로 이동

#### 2.6.2 반응형 디자인
- 모바일, 태블릿, 데스크톱 모든 디바이스에서 최적화된 경험 제공
- 터치 인터페이스 지원

## 3. UI/UX 컨셉 및 디자인 의도

### 3.1 전체 디자인 컨셉

#### 3.1.1 디자인 철학
- **모던하고 미니멀한 디자인**: 불필요한 요소를 제거하고 핵심 콘텐츠에 집중
- **부드럽고 친근한 느낌**: 과도하게 딱딱하지 않은, 접근하기 쉬운 분위기
- **시각적 매력**: 공유하고 싶게 만드는 아름다운 결과 페이지
- **직관적인 인터페이스**: 설명 없이도 사용할 수 있는 자명한 UI

#### 3.1.2 컬러 팔레트 방향성
- **메인 컬러**: 보라색-파란색 계열의 그라디언트 (신뢰감과 창의성)
- **배경 컬러**: 밝은 회색 또는 흰색 (깔끔하고 읽기 편한 환경)
- **텍스트 컬러**: 진한 회색 (눈의 피로도 최소화)
- **강조 컬러**: 메인 컬러와 대비되는 포인트 컬러

#### 3.1.3 타이포그래피 방향성
- **가독성 우선**: 명확하고 읽기 쉬운 폰트 선택
- **계층 구조**: 크기와 굵기로 정보의 중요도 구분
- **여백 활용**: 충분한 행간과 자간으로 편안한 읽기 경험

---

### 3.2 페이지별 디자인 의도

#### 3.2.1 시작 페이지 (랜딩 페이지)

**디자인 목표:**
- 첫 인상에서 신뢰감과 흥미를 동시에 전달
- 테스트 시작에 대한 심리적 장벽 낮추기

**주요 요소:**
- **히어로 섹션**: 서비스 이름과 간단한 설명을 중앙에 크게 배치
- **CTA 버튼**: "테스트 시작하기" 버튼을 눈에 띄게 강조 (그라디언트 효과, 큰 크기)
- **신뢰 지표**: 소요 시간, 참여자 수 등을 표시하여 안정감 제공
- **미니멀한 구성**: 산만하지 않게 핵심 요소만 배치

**시각적 효과:**
- 부드러운 그림자 효과로 깊이감 부여
- 호버 시 미묘한 애니메이션 (버튼 상승 효과 등)

---

#### 3.2.2 테스트 페이지

**디자인 목표:**
- 사용자가 테스트에 몰입할 수 있도록 방해 요소 최소화
- 진행 상황을 명확히 인지할 수 있도록 함
- 답변 선택이 쉽고 명확하도록 함

**주요 요소:**
- **프로그레스 바**: 페이지 최상단에 고정, 진행률과 질문 번호 표시
- **질문 카드**: 흰색 배경에 그림자 효과로 부각, 중앙 정렬
- **답변 옵션**: 충분히 큰 클릭 영역, 선택 시 색상 변화로 피드백
- **네비게이션**: 이전/다음 버튼을 질문 하단에 배치

**인터랙션:**
- 답변 선택 시 부드러운 색상 전환 애니메이션
- 질문 전환 시 페이드 인/아웃 효과
- 호버 시 옵션 강조

---

#### 3.2.3 결과 페이지

**디자인 목표:**
- 결과를 시각적으로 매력적이고 공유하고 싶게 만들기
- 많은 정보를 체계적으로 정리하여 가독성 확보
- SNS 공유에 최적화된 레이아웃

**주요 요소:**

1. **헤더 섹션**
   - MBTI 타입을 아주 크고 굵게 표시 (시각적 임팩트)
   - 그라디언트 배경으로 특별함 강조
   - 유형 별명을 부제로 배치

2. **특징 섹션**
   - 읽기 편한 문단 형태로 전체적인 성격 설명
   - 핵심 특징은 불릿 포인트로 간결하게 정리

3. **강점/약점 섹션**
   - 2단 레이아웃으로 좌우 배치 (데스크톱)
   - 아이콘과 함께 시각화
   - 각 항목을 간결하게 표현

4. **추천 직업 섹션**
   - 태그 형태로 나열 (유연한 배치)
   - 카테고리별 색상 구분 (선택 사항)

5. **유명인 섹션**
   - 프로필 이미지와 이름 병기 (선택 사항)
   - 태그 형태로 간결하게 나열

6. **액션 버튼 섹션**
   - 3개의 주요 버튼 (이미지 저장, 공유하기, 다시 하기)
   - 버튼 크기와 색상으로 중요도 구분
   - 아이콘과 텍스트 병기

7. **통계 섹션**
   - 차트를 활용한 시각적 통계 표시
   - 사용자가 다른 유형과 비교할 수 있도록 함

**시각적 효과:**
- 섹션 간 구분을 위한 경계선 또는 여백
- 스크롤 시 나타나는 페이드 인 애니메이션 (선택 사항)
- 카드형 레이아웃으로 정보 블록 구분

### 3.3 반응형 디자인 전략

#### 3.3.1 모바일 (768px 이하)
- 단일 컬럼 레이아웃
- 터치에 최적화된 버튼 크기 (최소 44x44px)
- 좌우 여백 최소화로 화면 공간 활용
- 강점/약점 섹션을 상하로 배치

#### 3.3.2 태블릿 (769px ~ 1024px)
- 질문 카드의 여백 증가
- 2단 레이아웃 부분 적용 (강점/약점 등)

#### 3.3.3 데스크톱 (1025px 이상)
- 최대 너비 제한으로 가독성 확보
- 2단 레이아웃 적극 활용
- 풍부한 여백으로 시각적 안정감

---

### 3.4 인터랙션 디자인

#### 3.4.1 애니메이션 원칙
- **부드러움**: 모든 전환은 0.2~0.3초의 부드러운 애니메이션
- **피드백**: 사용자 행동에 즉각적인 시각적 피드백 제공
- **성능**: 과도한 애니메이션으로 성능 저하 방지

#### 3.4.2 마이크로 인터랙션
- 버튼 호버 시 살짝 위로 이동
- 답변 선택 시 체크 마크 애니메이션
- 프로그레스 바 증가 애니메이션
- 공유/저장 완료 시 토스트 메시지

---

### 3.5 접근성 고려사항

#### 3.5.1 색상 접근성
- 충분한 색상 대비 (WCAG AA 기준 이상)
- 색상만으로 정보를 전달하지 않음 (아이콘, 텍스트 병기)

#### 3.5.2 키보드 네비게이션
- Tab 키로 모든 인터랙티브 요소 접근 가능
- Enter/Space 키로 선택 가능
- 포커스 상태 명확하게 표시

#### 3.5.3 스크린 리더 지원
- 시맨틱 HTML 사용
- ARIA 레이블 적절히 추가
- 이미지에 alt 텍스트 제공

## 4. 사용자 플로우

### 4.1 전체 사용자 여정
```
[랜딩 페이지 진입]
      ↓
[테스트 소개 확인]
      ↓
[테스트 시작 버튼 클릭]
      ↓
[질문 1/12 응답]
      ↓
[자동으로 다음 질문 진행]
      ↓
... (12개 질문 반복)
      ↓
[질문 12/12 완료]
      ↓
[결과 계산 중 (로딩)]
      ↓
[결과 페이지 표시]
      ↓
[결과 확인 및 읽기]
      ↓
[이미지 저장 또는 공유] (선택)
      ↓
[통계 확인] (선택)
      ↓
[다시 테스트하기] 또는 [페이지 종료]
```

### 4.2 주요 사용 시나리오

#### 시나리오 1: 처음 방문하는 사용자
1. 검색 또는 공유 링크를 통해 랜딩 페이지 진입
2. 서비스 소개와 참여자 수를 확인하며 신뢰감 형성
3. "테스트 시작하기" 버튼 클릭
4. 12개 질문에 순차적으로 답변 (약 3분 소요)
5. 프로그레스 바를 통해 진행 상황 확인
6. 결과 페이지에서 자신의 MBTI 유형과 상세 정보 확인
7. 결과가 마음에 들어 이미지로 저장하거나 SNS에 공유
8. 통계 차트에서 다른 사람들의 유형 분포 확인
9. 페이지를 나가거나 친구에게 공유

#### 시나리오 2: 재방문 사용자
1. 이전에 테스트를 완료한 사용자가 다시 방문
2. "다시 테스트하기"를 통해 재테스트 진행
3. 이전과 다른 결과가 나올 경우 비교
4. 통계에서 자신의 유형 비율 변화 확인

#### 시나리오 3: 공유 링크로 접속한 사용자
1. 친구가 공유한 링크를 통해 페이지 진입
2. 랜딩 페이지 또는 결과 페이지 도착
3. 친구의 결과를 보고 흥미를 느낌
4. "나도 테스트하기" 버튼 클릭
5. 테스트 진행 후 자신의 결과 확인
6. 친구와 결과 비교 및 대화의 소재로 활용

## 5. 성공 지표 (KPI)

### 5.1 사용자 참여 지표
- **테스트 완료율**: 테스트를 시작한 사용자 중 끝까지 완료한 비율 (목표: 80% 이상)
- **평균 테스트 시간**: 사용자가 테스트를 완료하는 데 걸리는 평균 시간 (목표: 3-5분)
- **공유율**: 테스트 완료 후 결과를 공유한 사용자 비율 (목표: 30% 이상)
- **재방문율**: 일정 기간 내 다시 방문한 사용자 비율 (목표: 20% 이상)

### 5.2 트래픽 지표
- **일일 활성 사용자 (DAU)**: 하루 동안 사이트를 방문한 고유 사용자 수 (목표: 500명)
- **월간 활성 사용자 (MAU)**: 한 달 동안 사이트를 방문한 고유 사용자 수 (목표: 10,000명)
- **바이럴 계수**: 한 사용자가 평균적으로 몇 명의 신규 사용자를 유입시키는지 (목표: 1.2 이상)

### 5.3 기술 성능 지표
- **페이지 로딩 시간**: 초기 페이지 로딩 완료 시간 (목표: 3초 이내)
- **모바일 성능 점수**: Lighthouse 모바일 성능 점수 (목표: 90점 이상)
- **에러율**: 사용자가 경험하는 오류 발생 비율 (목표: 1% 미만)

### 5.4 사용자 만족도 지표
- **결과 정확도 만족도**: 사용자가 결과에 공감하는 정도 (선택적 설문)
- **추천 의향**: 친구에게 추천할 의향이 있는 사용자 비율 (선택적 설문)

## 6. 향후 확장 가능성

### 6.1 단기 확장 (1-3개월)
- 다국어 지원 (영어) 추가
- 질문 수 확대로 정확도 향상 (12개 → 16개 또는 20개)

- A/B 테스트를 통한 UI/UX 최적화

### 6.2 중기 확장 (3-6개월)
- 백엔드 서버 구축 및 데이터베이스 도입

- 테스트 이력 저장 및 변화 추적
- 유형별 궁합 기능 (친구와의 궁합 분석)
- 유형별 추천 콘텐츠 (영화, 책, 음악 등)

### 6.3 장기 확장 (6개월 이상)
- 모바일 앱 개발 (iOS, Android)
- AI 기반 개인화된 조언 및 추천
- 커뮤니티 기능 (유형별 게시판, 채팅)
- 전문가 코멘트 및 심화 분석 (유료 서비스)
- 기업용 팀 분석 도구

- **대응**: 크로스 브라우저 테스트 철저히 수행, 폴리필 활용

- **대응**: 주기적으로 오래된 데이터 정리, 중요 데이터는 백엔드 마이그레이션 고려

- **대응**: 모든 콘텐츠 자체 제작, 일반적인 성격 유형 개념 활용, 필요 시 법률 검토

**리스크**: 개인정보 수집 관련 규제
- **대응**: 개인정보를 수집하지 않는 방식으로 설계, 익명 통계만 활용

## 8. 결론

이 PRD는 MBTI 성격 유형 테스트 웹사이트의 비전, 목표, 핵심 기능, 그리고 디자인 의도를 정리한 문서입니다.

**핵심 가치:**
- 사용자에게 즐겁고 의미 있는 자기 이해 경험 제공
- 직관적이고 매력적인 디자인으로 높은 완료율 달성
- 공유 기능을 통한 자연스러운 확산

이 제품은 간단하면서도 몰입도 높은 경험을 제공하여 사용자가 자신의 성격을 이해하고, 그 결과를 친구들과 공유하며 즐거움을 느낄 수 있도록 설계되었습니다. 향후 사용자 피드백을 바탕으로 점진적으로 기능을 확장하고 개선해 나갈 계획입니다.
